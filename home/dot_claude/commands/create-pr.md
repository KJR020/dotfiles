# Create Pull Request

現在のブランチとマージ先ブランチのdiffを分析し、PR作成用のタイトルと本文を生成します。

## 実行手順

1. 現在のブランチ名を取得
2. マージ先ブランチを特定（通常は `main` または `master`）
3. `git diff <base-branch>...HEAD` でdiffを取得
4. コミット履歴を取得 `git log <base-branch>..HEAD`
5. 変更内容を分析
6. 以下の形式でPRタイトルと本文を生成

## 出力形式

```markdown
### タイトル
[簡潔で分かりやすいタイトル]

### 本文
- 概要
  - [このPRの目的や背景を1-2文で]
  - [解決する課題や追加する機能]

- 実装内容
  - [主要な変更点1]
  - [主要な変更点2]
  - [主要な変更点3]

- テスト
  - [テストの有無や確認方法]
```

## 注意事項

- タイトルは50文字以内を推奨
- 概要は「なぜ」を説明
- 実装内容は「何を」を説明
- 技術的な詳細は適度に抽象化
- ファイル数が多い場合は主要な変更に絞る

## 実行後

生成されたタイトルと本文を以下の形式で表示します：

```bash
gh pr create --title "<タイトル>" --body "<本文>"
```

## 注記

このコマンドはPRのタイトルと本文の生成、および `gh pr create` コマンドの提案のみを行います。実際のPR作成はユーザー自身が上記のコマンドを実行して行ってください。
